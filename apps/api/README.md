# apps/api

Geplanter API-Service für Business-Logik, Agenten-Orchestrierung und kontrollierte Tool-Nutzung.

Der spätere Verantwortungsbereich umfasst:

- strukturierte Agenten-Calls
- Workflow-nahe Geschäftslogik
- Freigabe- und Zustandsregeln
- Audit-Logging
- Integrationsadapter

Die API wird bewusst erst dann implementiert, wenn Zustandsmodell, Event-Modell und Schnittstellengrenzen finalisiert sind.
